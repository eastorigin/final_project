나이, 우편번호: int
	- int 타입으로 연산이 가능한가?
전화번호: 010-1234-1234 <-- String


1. 사람정보 (클래스)
	- 멤버변수들
		- 이름 (String)
		- 나이 (int)
		- 생년월일 (LocalDate)
		- 주소 (데이터 클래스) - 사람정보 has a 주소
		- 별명 (String)
		- 연락처 (String)
		- 이메일 (String)
		- 추가날짜 (LocalDate)
		- 마지막 수정날짜 (LocalDate)
		
2. 주소 (클래스)
	- 멤버변수들
		- 시/군/구 (String)
		- 읍/면/동 (String)
		- 길 (도로명주소) (String)
		- 번지 (구 주소) (String)
		- 동/호수 (상세주소) (String)
		- 우편번호 (String) // 연산이 의미없어서 int가 아님
		
3. 주소록 (클래스)
	- 멤버변수들
		- 사람 정보들 (List<사람정보>) - 주소록 has a 사람정보
		// 배열은 크기 제한이 있어서 안 되고, 맵은 키가 이름일 경우 변경될 수 있어서 부가
		
	- 기능들
		- 이름 + 별명 검색 (String 검색어)
			:: 파라미터로 들어온 검색어가 이름 혹은 별명에 포함되어 있다면, 해당 사람정보의 인덱스와 이름 (나이) 를 출력한다
				검색어가 비어있으면 모든 사람 정보의 인덱스와 이름(나이)를 출력한다
				검색 결과가 없을 때에는 "해당하는 사람이 없습니다"를 출력한다
		- 연락처 검색 (String 검색어)
			:: 파라미터로 들어온 검색어가 연락처에 포함되어 있다면, 해당 사람정보의 인덱스와 이름 및 연락처를 출력한다
				검색어가 비어있으면 모든 사람 정보의 인덱스와 이름, 연락처를 출력한다
				검색 결과가 없을 때에는 "등록된 연락처가 없습니다"를 출력한다
		- 보름 이내에 생일을 맞이하는 사람 검색 () // 기준이 오늘이라 어차피 LocalDate.now()하면 된다 그래서 파라미터가 있으면 안 된다
			:: 기준을 당일로 하고 등록된 각 사람정보의 생일이 오늘보다 15일 이내로 after이면, 해당 사람정보의 인덱스와 이름, 생일을 출력한다
				검색 결과가 없을 때에는 "앞으로 보름 이내에 생일인 사람이 없습니다"라고 출력한다
		- 보름 전에 생일을 맞이한 사람 검색
			:: 당일 기준으로 각 사람정보의 생일이 15일 이내로 before 이면 해당 사람정보의 인덱스와, 이름, 생일을 출력한다
				검색 결과가 없을 때에는 "과거 보름 동안 생일이었던 사람이 없습니다"를 출력한다
		- 주소 검색 (String 검색어)
			- 주소 전체 정보를 검색
				- toString() 활용
					:: 파라미터로 들어온 검색어가 주소 전체 정보 중 하나라도 포함되어 있다면, 해당 사람정보의 인덱스와 이름 및 전체 주소 정보를 출력한다
						검색어가 비어있으면 모든 사람 정보의 인덱스와 이름, 주소 전체 정보를 출력한다
						검색 결과가 없을 때에는 "일치하는 주소가 없습니다"라고 출력한다
		- 등록 - 사람정보들에 새로운 주소록을 추가 (사람정보 등록_인스턴스)
			:: 파라미터 정보를 사람 정보들 List에 추가한다
				등록 인스턴스가 null이거나 이름이 비어있다면 "등록할 수 없습니다" 예외를 던진다
		- 등록 - 사람정보들에 새로운 주소록을 추가 (이름, 나이, 생년월일, 주소, 별명, 연락처, 이메일, 추가날짜, 수정날짜) // 복잡함
		- 한 사람 조회 - 사람정보들에서 한 사람 정보만 조회 (int 인덱스)
			:: 파라미터로 들어온 인덱스에 해당되는 한 사람의 모든 정보를 출력한다
				빈 값을 입력하면 "숫자를 입력해주세요"를 출력한다
				해당 인덱스에 사람 정보가 없으면 "해당 인덱스로 등록된 사람이 없습니다"라고 출력한다
				숫자가 아닌 다른 값을 입력하면 "숫자를 정확히 입력해주세요"를 출력한다
		- 삭제 - 사람정보들에서 특정인만 삭제 (int 인덱스)
			:: 파라미터로 들어온 인덱스에 해당되는 사람의 모든 정보를 삭제한다
				삭제하기 전에 재확인 질문을 한다
				해당 인덱스에 등록된 사람이 없으면 "등록된 정보가 없습니다"라고 출력한다
				숫자가 아닌 다른 값을 입력하면 "정확한 숫자를 입력하세요"라고 출력한다
		- 수정 - 사람정보들에서 특정인만 수정 (int 인덱스, 사람정보 새로운 사람정보)
			:: 파라미터로 들어온 인덱스에 해당되는 사람의 정보를 파라미터로 들어온 새로운 사람정보로 교체한다
				수정하기 전에 재확인 질문을 한다
				새로운 사람정보 기존의 추가날짜를 할당하고 마지막 수정날짜를 오늘로 할당한다
				해당 인덱스에 등록된 사람이 없으면 "등록된 정보가 없습니다"라고 출력한다
				숫자가 아닌 다른 값을 입력하면 "정확한 숫자를 입력하세요"라고 출력한다
